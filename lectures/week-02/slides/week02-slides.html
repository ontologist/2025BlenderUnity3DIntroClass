🎭 What are Materials?

The Rendering Equation

Materials + Lighting = Final Appearance

Materials define:

  • How surfaces appear
  • Color properties
  • Shininess / Roughness
  • Transparency
  • How light interacts with surface

マテリアルが定義するもの:

  • 表面の見え方
  • 色のプロパティ
  • 光沢 / ラフネス
  • 透明度
  • 光と表面の相互作用

⚙️ Material Properties

Property / プロパティ Description / 説明 Values / 値
Base Color Fundamental surface color
基本的な表面色		 RGB color picker
Metallic Metal vs non-metal
金属vs非金属		 0.0 (plastic) to 1.0 (metal)
Roughness Shiny vs matte
光沢vsマット		 0.0 (mirror) to 1.0 (matte)
Alpha Transparency
透明度		 0.0 (invisible) to 1.0 (opaque)

🔧 Shader Editor Overview

Accessing Shader Editor:

  • Switch to Shading workspace (top menu)
  • Node-based system
  • Boxes connected by lines

Key Nodes:

  • Principled BSDF: Industry-standard shader
  • Material Output: Final render connection
  • Image Texture: For applying textures

Shader Editorへのアクセス:

  • Shadingワークスペースに切り替え（上部メニュー）
  • ノードベースシステム
  • 線で接続されたボックス

主要なノード:

  • Principled BSDF: 業界標準シェーダー
  • Material Output: 最終レンダー接続
  • Image Texture: テクスチャ適用用

🖌️ Applying Materials: Step-by-Step

Process / プロセス:
  1. Select Object
    • Click in viewport or outliner
  2. Add Material
    • Shading workspace
    • Material Properties (sphere icon)
    • Click "+ New"
  3. Adjust Properties
    • Base Color → Pick color
    • Adjust Metallic (0-1)
    • Adjust Roughness (0-1)
  4. View Results
    • Material Preview mode (3rd sphere)
    • Or Rendered view (4th sphere)

👁️ Viewport Shading Modes

Mode Shortcut / ショートカット Description / 説明
Wireframe Z → 1st sphere Shows mesh structure only
メッシュ構造のみ表示
Solid Z → 2nd sphere Basic shading, no materials
基本シェーディング、マテリアルなし
Material Preview Z → 3rd sphere Shows materials with studio lighting
スタジオライティングでマテリアルを表示
Rendered Z → 4th sphere Full render with scene lighting
シーンライティングで完全レンダー

💡 Tip: Use Material Preview for quick material work, Rendered for final results

📐 What is UV Unwrapping?

The Concept:

"Unfolding" a 3D model into a 2D layout

Think of it like: Unwrapping a gift box and laying the paper flat

Why UV Unwrapping?

  • To apply 2D images (textures) onto 3D surfaces
  • UV = 2D coordinates (U and V axes)
  • Maps image pixels to 3D surface points

概念:

3Dモデルを2Dレイアウトに「展開」

例えると: ギフトボックスを開いて紙を平らにする

なぜUV展開？

  • 2D画像（テクスチャ）を3D表面に適用するため
  • UV = 2D座標（UとV軸）
  • 画像ピクセルを3D表面ポイントにマップ

🔪 UV Unwrapping Process

Steps / 手順:
  1. Enter Edit Mode
    • Tab
  2. Select All Faces
    • A (select all)
  3. Mark Seams
    • Select edges where to "cut"
    • Ctrl + E → Mark Seam
  4. Unwrap
    • U → Unwrap
  5. View UV Layout
    • Switch to UV Editing workspace

💡 Where to Place Seams: Sharp corners, hidden areas, natural breaks

🎬 Introduction to Animation

What is Animation?

Creating the illusion of movement by displaying a sequence of images (frames) over time

Key Concepts:

  • Frame Rate: 24-30 frames per second (fps)
  • Timeline: Shows frame range
  • Keyframes: Important animation positions
  • Interpolation: Automatic in-between frames

アニメーションとは？

時間の経過とともに画像（フレーム）のシーケンスを表示することで動きの錯覚を作成

主要な概念:

  • フレームレート: 24〜30フレーム/秒（fps）
  • タイムライン: フレーム範囲を表示
  • キーフレーム: 重要なアニメーション位置
  • 補間: 自動的な中間フレーム

🛤️ Path-Based Animation

Concept:

Object follows a predefined curve or path

Use Cases:

  • Camera fly-throughs
  • Moving platforms in games
  • Patrol routes for enemies
  • Orbiting objects
  • Collectible items

使用例:

  • カメラのフライスルー
  • ゲーム内の移動プラットフォーム
  • 敵のパトロールルート
  • 軌道上のオブジェクト
  • 収集可能なアイテム

🎯 Creating Path Animation

Step-by-Step / ステップバイステップ:
  1. Create Path:
    • Shift + A → Curve → Bezier Curve
    • Edit Mode → Move control points to shape path
  2. Create Object:
    • Add sphere or any object to animate
  3. Add Follow Path Constraint:
    • Select object
    • Constraints → Add → Follow Path
    • Target: Select curve
    • Check "Follow Curve"
  4. Set Animation:
    • Select curve
    • Object Data Properties → Path Animation
    • Set frame range (e.g., 0-120)

⌨️ Animation Shortcuts

Shortcut Action / アクション Description / 説明
Spacebar Play/Pause Start or stop animation playback
アニメーション再生を開始または停止
← → Frame Step Move one frame forward/backward
1フレーム前後に移動
Shift + ← First Frame Jump to first frame
最初のフレームにジャンプ
Shift + → Last Frame Jump to last frame
最後のフレームにジャンプ

🦴 Introduction to Bone Rigging

What is Rigging?

Creating a skeleton system (bones/armature) that controls mesh deformation

Key Concepts:

  • Armature: Collection of bones
  • Bones: Individual control elements
  • Skinning: Binding mesh to bones
  • Pose Mode: Posing and animating bones

リギングとは？

メッシュの変形を制御する骨格システム（ボーン/アーマチュア）の作成

主要な概念:

  • アーマチュア: ボーンのコレクション
  • ボーン: 個々の制御要素
  • スキニング: メッシュをボーンにバインド
  • ポーズモード: ボーンのポージングとアニメーション

📌 Note: Rigging is complex! This is just an introduction. We'll use pre-rigged characters later.

🔗 Basic Rigging Process

Quick Overview / 簡単な概要:
  1. Add Armature:
    • Shift + A → Armature → Single Bone
  2. Edit Bones (Edit Mode):
    • Tab → Edit Mode
    • Move bone head/tail
    • E to extrude and add bones
  3. Parent Mesh to Armature:
    • Select mesh, then armature
    • Ctrl + P → With Automatic Weights
  4. Pose Mode:
    • Select armature
    • Switch to Pose Mode
    • R / G to rotate/move bones

📦 Exporting for Unity

Export Format: FBX

Process:

  1. File → Export → FBX (.fbx)
  2. Check "Selected Objects" (if needed)
  3. Apply Modifiers: ON
  4. Choose location and filename
  5. Click "Export FBX"

Pre-Export Checklist:

  • ✅ Materials applied
  • ✅ Textures in project folder
  • ✅ Objects properly named
  • ✅ Appropriate scale

エクスポート形式: FBX

プロセス:

  1. File → Export → FBX（.fbx）
  2. 「Selected Objects」にチェック（必要な場合）
  3. Apply Modifiers: ON
  4. 場所とファイル名を選択
  5. 「Export FBX」をクリック

エクスポート前チェックリスト:

  • ✅ マテリアルが適用されている
  • ✅ テクスチャがプロジェクトフォルダにある
  • ✅ オブジェクトが適切に名前付けされている
  • ✅ 適切なスケール

⚠️ Common Pitfalls to Avoid

Materials Issues:

  • ❌ Not showing → Check viewport shading
  • ❌ Black material → Check lighting

Texture Issues:

  • ❌ Stretched texture → Re-unwrap with seams
  • ❌ Missing texture → Pack resources

Animation Issues:

  • ❌ Not playing → Check frame range
  • ❌ Object not following → Check constraint target

マテリアルの問題:

  • ❌ 表示されない → ビューポートシェーディングを確認
  • ❌ 黒いマテリアル → ライティングを確認

テクスチャの問題:

  • ❌ 引き伸ばされたテクスチャ → シームで再展開
  • ❌ 欠けているテクスチャ → リソースをパック

アニメーションの問題:

  • ❌ 再生されない → フレーム範囲を確認
  • ❌ オブジェクトが従わない → 制約ターゲットを確認
