🎯 What is a GameObject?

Everything in Unity is a GameObject!

  • Container for Components
  • Can be visible or invisible
  • Can be parent or child
  • Always has a Transform

Examples:

  • 3D models (cube, sphere)
  • Lights
  • Cameras
  • Empty objects (folders)
  • UI elements
  • Audio sources

UnityのすべてがGameObject！

  • Componentのコンテナ
  • 可視または不可視
  • 親または子になれる
  • 常にTransformを持つ

例：

  • 3Dモデル（立方体、球）
  • ライト
  • カメラ
  • 空のオブジェクト（フォルダ）
  • UI要素
  • オーディオソース

📐 Transform Component

Position / 位置

  • X: Left (-) / Right (+)
  • Y: Down (-) / Up (+)
  • Z: Back (-) / Forward (+)
  • Origin: 0, 0, 0

Rotation / 回転

  • X: Pitch (tilt forward/back) / ピッチ
  • Y: Yaw (turn left/right) / ヨー
  • Z: Roll (tilt sideways) / ロール
  • Measured in degrees / 度単位で測定

Scale / スケール

  • 1, 1, 1: Normal size / 通常サイズ
  • 2, 2, 2: Twice as large / 2倍の大きさ
  • 0.5, 0.5, 0.5: Half size / 半分のサイズ
  • Non-uniform: (2, 1, 1) = stretched on X

🌳 Hierarchy & Parent-Child Relationships

Organizing with Hierarchy:

  • Drag child object onto parent
  • Child inherits parent's transform
  • Move parent → children move too
  • Rotate parent → children rotate too
  • Delete parent → children deleted too

Best Practices:

  • Create empty GameObjects as folders
  • Example: "Environment", "Props", "Lighting"
  • Group related objects together
  • Name objects clearly

Hierarchyで整理:

  • 子オブジェクトを親にドラッグ
  • 子は親のトランスフォームを継承
  • 親を移動 → 子も移動
  • 親を回転 → 子も回転
  • 親を削除 → 子も削除

ベストプラクティス:

  • フォルダとして空のGameObjectを作成
  • 例：「Environment」、「Props」、「Lighting」
  • 関連オブジェクトをグループ化
  • オブジェクトを明確に命名

📦 Introduction to Prefabs

What are Prefabs?

Reusable GameObject templates

再利用可能なGameObjectテンプレート

Why use Prefabs?

  • Create once, use many times
  • Change prefab → all instances update
  • Saves time in level design
  • Consistent objects across scenes
  • Easy to manage and modify

Examples:

  • Wall segments in a maze
  • Trees in a forest
  • Enemies in a game
  • Collectible items

なぜPrefabを使うのか？

  • 一度作成して何度も使用
  • Prefabを変更 → すべてのインスタンスが更新
  • レベルデザインの時間を節約
  • シーン間で一貫したオブジェクト
  • 管理と変更が簡単

例：

  • 迷路の壁セグメント
  • 森の木
  • ゲームの敵
  • 収集アイテム

📦 Creating & Using Prefabs

Step-by-Step / ステップバイステップ:

1. Create a GameObject
- Design and configure it in the scene
- Add all necessary components

2. Drag to Project Window
- Drag from Hierarchy to Project
- Choose "Original Prefab"
- Prefab asset created!

3. Use the Prefab
- Drag from Project to Scene
- Creates new instance
- Blue text = Prefab instance

4. Edit the Prefab
- Double-click prefab asset
- Enter Prefab Mode
- Changes affect all instances

🔍 Inspector Window / インスペクターウィンドウ

What does it show?

  • All components on selected GameObject
  • Component properties and values
  • Enable/disable components (checkbox)
  • Add/remove components

Key Features:

  • Add Component: Button at bottom
  • Gear icon: Component menu
  • Checkbox: Enable/disable
  • Reset: Reset to default values
  • Remove: Delete component

何を表示？

  • 選択したGameObjectのすべてのコンポーネント
  • コンポーネントのプロパティと値
  • コンポーネントの有効/無効（チェックボックス）
  • コンポーネントの追加/削除

主要機能：

  • コンポーネントを追加: 下部のボタン
  • 歯車アイコン: コンポーネントメニュー
  • チェックボックス: 有効/無効
  • リセット: デフォルト値にリセット
  • 削除: コンポーネントを削除

📤 Exporting from Blender to Unity

Blender Export Settings:

1. Select your model(s) in Blender
2. File → Export → FBX (.fbx)

Important Settings:
✓ Selected Objects: ON (if only exporting selected)
✓ Scale: 1.00
✓ Forward: -Z Forward
✓ Up: Y Up
✓ Apply Scalings: FBX All

3. Save to Unity's Assets folder
4. Name clearly: Maze_Week01.fbx
⚠️ Important! Blender uses Z-up, Unity uses Y-up.
These settings fix the orientation!
⚠️ 重要！ BlenderはZ-up、UnityはY-upを使用。
これらの設定で向きを修正！

📥 Importing FBX to Unity

Import Methods:

  1. Drag & Drop: Drag .fbx into Project window
  2. Copy: Copy .fbx to Assets folder

Import Settings:

  • Select imported model in Project
  • Check Inspector for settings
  • Model tab: Scale, meshes
  • Materials tab: Material extraction
  • Click Apply after changes

Using the Model:

  • Drag from Project to Scene
  • Adjust Transform (position, rotation, scale)
  • Add materials for color/texture

インポート方法:

  1. ドラッグ＆ドロップ: .fbxをProjectウィンドウにドラッグ
  2. コピー: .fbxをAssetsフォルダにコピー

インポート設定:

  • Projectでインポートしたモデルを選択
  • Inspectorで設定を確認
  • Modelタブ: スケール、メッシュ
  • Materialsタブ: マテリアルの抽出
  • 変更後にApplyをクリック

モデルの使用:

  • ProjectからSceneにドラッグ
  • Transformを調整（位置、回転、スケール）
  • 色/テクスチャ用にマテリアルを追加

⚠️ Common Mistakes to Avoid

Navigation Issues:

  • ❌ Getting lost in Scene view
  • ✅ Use F to frame selected

Scale Problems:

  • ❌ Model too big/small
  • ✅ Check import scale settings

Organization:

  • ❌ Everything at root level
  • ✅ Use empty GameObjects as folders

Forgetting to Save:

  • ❌ Lose hours of work
  • ✅ Ctrl+S frequently!

ナビゲーションの問題:

  • ❌ Sceneビューで迷子
  • ✅ Fを使用して選択をフレーム

スケールの問題:

  • ❌ モデルが大きすぎる/小さすぎる
  • ✅ インポートスケール設定を確認

整理:

  • ❌ すべてがルートレベル
  • ✅ フォルダとして空のGameObjectを使用

保存を忘れる:

  • ❌ 何時間もの作業を失う
  • ✅ Ctrl+Sを頻繁に！
