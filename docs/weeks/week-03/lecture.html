Segment 3: GameObjects & Component Architecture

Understanding GameObjects / GameObjectの理解

English:

  • Everything in Unity is a GameObject
  • GameObjects are containers for Components
  • Empty by default (just a Transform)
  • Functionality comes from Components
  • Parent-child relationships for organization

日本語:

  • UnityのすべてはGameObject
  • GameObjectはComponentのコンテナ
  • デフォルトでは空（Transformのみ）
  • 機能はComponentから提供
  • 整理のための親子関係

Component System / コンポーネントシステム

English: Components define what a GameObject can do:

  • Transform: Position, rotation, scale (every GameObject has this)
  • Mesh Filter: Defines the 3D shape
  • Mesh Renderer: Makes the object visible
  • Collider: Enables physics collision detection
  • Rigidbody: Adds physics simulation (gravity, forces)
  • Light: Emits light in the scene
  • Camera: Renders what the player sees
  • Audio Source: Plays sounds
  • Scripts: Custom behavior (C# code)

日本語: Componentは、GameObjectができることを定義します：

  • Transform: 位置、回転、スケール（すべてのGameObjectにあります）
  • Mesh Filter: 3D形状を定義
  • Mesh Renderer: オブジェクトを可視化
  • Collider: 物理衝突検出を有効化
  • Rigidbody: 物理シミュレーション（重力、力）を追加
  • Light: シーンに光を放射
  • Camera: プレイヤーが見るものをレンダリング
  • Audio Source: サウンドを再生
  • Scripts: カスタム動作（C#コード）

🎯 Component Philosophy / コンポーネント哲学:

English: Instead of creating object classes (like "Enemy" or "Player"), Unity uses composition. You build objects by combining components. This makes Unity extremely flexible!

日本語: オブジェクトクラス（「Enemy」や「Player」など）を作成する代わりに、Unityは構成を使用します。コンポーネントを組み合わせてオブジェクトを構築します。これによりUnityは非常に柔軟になります！

Segment 4: Transform Component Deep Dive

Transform Properties / Transformプロパティ

Position / 位置

English:

  • X, Y, Z coordinates in 3D space
  • World space vs Local space
  • 0,0,0 is the origin/center of the world
  • Positive Y is up, positive X is right, positive Z is forward

日本語:

  • 3D空間のX、Y、Z座標
  • ワールド空間 vs ローカル空間
  • 0,0,0はワールドの原点/中心
  • 正のYは上、正のXは右、正のZは前方

Rotation / 回転

English:

  • Euler angles (X, Y, Z in degrees)
  • X rotation: pitch (tilt forward/back)
  • Y rotation: yaw (turn left/right)
  • Z rotation: roll (tilt sideways)
  • Watch out for gimbal lock!

日本語:

  • オイラー角（X、Y、Z度単位）
  • X回転：ピッチ（前後に傾ける）
  • Y回転：ヨー（左右に回す）
  • Z回転：ロール（横に傾ける）
  • ジンバルロックに注意！

Scale / スケール

English:

  • Size multiplier on each axis
  • 1,1,1 is normal/default size
  • 2,2,2 is twice as large
  • 0.5,0.5,0.5 is half size
  • Non-uniform scale possible (2,1,1 = stretched on X)

日本語:

  • 各軸のサイズ乗数
  • 1,1,1は通常/デフォルトサイズ
  • 2,2,2は2倍の大きさ
  • 0.5,0.5,0.5は半分のサイズ
  • 非均一スケール可能（2,1,1 = Xに引き伸ばす）

🎹 Transform Tool Shortcuts / Transformツールショートカット:

  • Q = Hand tool (pan view) / ハンドツール（ビューをパン）
  • W = Move tool / 移動ツール
  • E = Rotate tool / 回転ツール
  • R = Scale tool / スケールツール
  • T = Rect tool (2D) / Rectツール（2D）
  • Y = Combined transform tool / 結合トランスフォームツール
  • Ctrl + D = Duplicate selected / 選択を複製
  • Delete = Delete selected / 選択を削除

Gizmo Modes / ギズモモード

English: Toggle between Local and Global space with the button in the toolbar

  • Global: Axes aligned with world space
  • Local: Axes aligned with object's rotation

日本語: ツールバーのボタンでローカルとグローバル空間を切り替え

  • グローバル: ワールド空間に沿った軸
  • ローカル: オブジェクトの回転に沿った軸

Segment 5: Introduction to Prefabs

What are Prefabs? / Prefabとは？

English:

  • Reusable GameObject templates
  • Create once, use many times
  • Changes to prefab affect all instances
  • Can override individual properties
  • Essential for level design efficiency
  • Blue text in Hierarchy indicates prefab instance

日本語:

  • 再利用可能なGameObjectテンプレート
  • 一度作成して何度も使用
  • Prefabへの変更はすべてのインスタンスに影響
  • 個々のプロパティをオーバーライド可能
  • レベルデザインの効率に不可欠
  • Hierarchyの青いテキストはPrefabインスタンスを示す

Creating a Prefab / Prefabの作成

English - Steps:

  1. Create and configure a GameObject in the scene
  2. Drag it from Hierarchy to Project window
  3. Choose "Original Prefab" when prompted
  4. Prefab asset is created in Project
  5. Original becomes prefab instance (blue text)

日本語 - 手順:

  1. シーンでGameObjectを作成して設定
  2. HierarchyからProjectウィンドウにドラッグ
  3. プロンプトが表示されたら「Original Prefab」を選択
  4. ProjectにPrefabアセットが作成される
  5. 元のオブジェクトがPrefabインスタンスになる（青いテキスト）

Using Prefabs / Prefabの使用

  • Drag prefab from Project to Scene to create instances
  • Each instance shares the prefab's properties
  • Modify prefab: double-click to enter Prefab Mode
  • Apply overrides: Inspector → Overrides → Apply All
  • Revert changes: Overrides → Revert All

💡 Prefab Best Practices / Prefabのベストプラクティス:

  • Use prefabs for any object that appears multiple times
  • Create a "Prefabs" folder in your Project
  • Name prefabs clearly (Wall_Straight, Wall_Corner, etc.)
  • Use nested prefabs for complex objects

Segment 6: Importing Blender Models into Unity

Export from Blender / Blenderからエクスポート

English - Blender Export Steps:

  1. In Blender: Select your model(s)
  2. File → Export → FBX (.fbx)
  3. Export settings:
    • Selected Objects: ON (if you only want selected)
    • Scale: 1.00
    • Forward: -Z Forward
    • Up: Y Up
    • Apply Scalings: FBX All
  4. Save to your Unity project's Assets folder
  5. Name clearly: Maze_Week01.fbx

日本語 - Blenderエクスポート手順:

  1. Blenderで：モデルを選択
  2. ファイル → エクスポート → FBX (.fbx)
  3. エクスポート設定:
    • 選択したオブジェクト：ON（選択したもののみ必要な場合）
    • スケール：1.00
    • 前方：-Z Forward
    • 上：Y Up
    • スケーリングを適用：FBX All
  4. UnityプロジェクトのAssetsフォルダに保存
  5. 明確に命名：Maze_Week01.fbx

⚠️ Common Export Issues / よくあるエクスポートの問題:

  • Wrong size: Check scale settings in both Blender and Unity
  • Wrong orientation: Verify Forward/Up axis settings
  • Missing materials: Materials don't export with FBX (recreate in Unity)
  • Complex hierarchy: Apply transforms before export

Import to Unity / Unityにインポート

English - Unity Import Steps:

  1. Method 1: Drag .fbx file into Unity Project window
  2. Method 2: Copy .fbx to Assets folder (auto-imports)
  3. Select imported model in Project window
  4. Inspector shows import settings:
    • Model tab: Scale factor, meshes
    • Rig tab: Animation settings (none for now)
    • Animation tab: Animation clips (none for now)
    • Materials tab: Material extraction
  5. Click Apply after changing settings

日本語 - Unityインポート手順:

  1. 方法1: .fbxファイルをUnity Projectウィンドウにドラッグ
  2. 方法2: .fbxをAssetsフォルダにコピー（自動インポート）
  3. Projectウィンドウでインポートしたモデルを選択
  4. Inspectorにインポート設定を表示:
    • Modelタブ: スケールファクター、メッシュ
    • Rigタブ: アニメーション設定（今はなし）
    • Animationタブ: アニメーションクリップ（今はなし）
    • Materialsタブ: マテリアルの抽出
  5. 設定を変更した後にApplyをクリック

Using Imported Models / インポートされたモデルの使用

  • Drag model from Project to Scene or Hierarchy
  • Model appears as GameObject with Mesh Filter and Mesh Renderer
  • Adjust Transform as needed (position, rotation, scale)
  • Create materials in Unity for proper appearance
  • Consider making complex models into prefabs

💡 Model Organization Tips / モデル整理のヒント:

  • Create "Models" folder in Assets
  • Create "Materials" folder for Unity materials
  • Keep Blender source files in separate folder (not in Assets)
  • Use consistent naming: ModelName_LOD0.fbx

Common Pitfalls / よくある落とし穴

1. Wrong Coordinate System / 間違った座標系

Problem: Blender uses Z-up, Unity uses Y-up

Solution: Use correct export settings: Forward = -Z, Up = Y

2. Scale Inconsistencies / スケールの不一致

Problem: Model appears too large or too small in Unity

Solution: Apply scale in Blender before export, use Scale Factor in Unity import settings

3. Lost in Scene View / シーンビューで迷子

Problem: Can't find objects or navigate properly

Solution: Select object in Hierarchy, press F to frame it

4. Component Confusion / コンポーネントの混乱

Problem: Adding wrong components or too many components

Solution: Start simple - basic objects only need Transform, Mesh Filter, Mesh Renderer

5. Hierarchy Disorganization / ヒエラルキーの無秩序

Problem: All objects at root level, hard to find things

Solution: Use empty GameObjects as folders (Environment, Props, Lighting, etc.)

6. Forgetting to Save Scene / シーンの保存を忘れる

Problem: Losing work when Unity crashes

Solution: Ctrl+S frequently, enable Auto Save in preferences
