Assignment Description / 課題概要

English: This week, you'll transition from Blender to Unity by importing your Week 1 maze and setting up a complete Unity scene. You'll learn the Unity interface, work with GameObjects and Components, and prepare the foundation for a playable ball-maze game that will be completed in Week 4.

日本語: 今週は、Week 1の迷路をインポートし、完全なUnityシーンをセットアップすることで、BlenderからUnityへ移行します。Unityインターフェースを学び、GameObjectとComponentを操作し、Week 4で完成するプレイ可能なボール迷路ゲームの基礎を準備します。

Requirements / 要件

1. Export Maze from Blender 3 points

English:

  • Open your Week 1 maze in Blender
  • Apply all transforms (Ctrl+A → All Transforms)
  • Export as FBX with correct settings:
    • Forward: -Z Forward
    • Up: Y Up
    • Apply Scalings: FBX All
    • Scale: 1.00
  • File name: Maze_Week01.fbx
  • Save to Unity project's Assets/Models folder

日本語:

  • BlenderでWeek 1の迷路を開く
  • すべてのトランスフォームを適用（Ctrl+A → All Transforms）
  • 正しい設定でFBXとしてエクスポート:
    • 前方：-Z Forward
    • 上：Y Up
    • スケーリングを適用：FBX All
    • スケール：1.00
  • ファイル名：Maze_Week01.fbx
  • UnityプロジェクトのAssets/Modelsフォルダに保存

⚠️ Important Export Tips:

  • Make sure all objects are visible before export
  • Check that scale is applied in Blender (Object Properties → Scale should be 1,1,1)
  • Export only the maze objects (select them first, enable "Selected Objects")

2. Set Up Unity Project & Import Model 3 points

English:

  • Create new Unity 3D Core project: GameDev2025_Week03
  • Save in your GitHub repository folder: week-03/
  • Create folder structure in Assets:
    • Models
    • Materials
    • Prefabs
    • Scenes
  • Import FBX file (drag into Models folder)
  • Check import settings in Inspector
  • Adjust Scale Factor if needed (usually 1 or 100)
  • Click Apply

日本語:

  • 新しいUnity 3D Coreプロジェクトを作成：GameDev2025_Week03
  • GitHubリポジトリフォルダに保存：week-03/
  • Assetsにフォルダ構造を作成:
    • Models
    • Materials
    • Prefabs
    • Scenes
  • FBXファイルをインポート（Modelsフォルダにドラッグ）
  • Inspectorでインポート設定を確認
  • 必要に応じてScale Factorを調整（通常1または100）
  • Applyをクリック

💡 Scale Troubleshooting:

If your maze appears too large or too small, select the imported FBX in Project window, go to Inspector → Model tab → Scale Factor → Try values like 0.01, 0.1, 1, or 100 until it looks right.

3. Build the Scene 4 points

English:

  • Create new scene: MazeGame
  • Save in Scenes folder
  • Drag imported maze model into scene
  • Position at (0, 0, 0)
  • Add Ball GameObject:
    • Create 3D Object → Sphere
    • Rename to "Ball"
    • Scale to 0.5, 0.5, 0.5 (or appropriate size for maze)
    • Position above maze start area
  • Configure Main Camera:
    • Position to see entire maze
    • Angle looking down at maze (rotate around X-axis)
    • Recommended: Position (0, 15, -10), Rotation (45, 0, 0)
  • Configure Directional Light:
    • Adjust rotation for good shadows
    • Recommended: Rotation (50, -30, 0)
    • Intensity: 1.0

日本語:

  • 新しいシーンを作成：MazeGame
  • Scenesフォルダに保存
  • インポートした迷路モデルをシーンにドラッグ
  • 位置を(0, 0, 0)に設定
  • Ball GameObjectを追加:
    • 3D Object → Sphereを作成
    • 「Ball」にリネーム
    • スケールを0.5, 0.5, 0.5（または迷路に適したサイズ）に設定
    • 迷路のスタートエリアの上に配置
  • Main Cameraを設定:
    • 迷路全体が見える位置に配置
    • 迷路を見下ろす角度（X軸周りに回転）
    • 推奨：位置(0, 15, -10)、回転(45, 0, 0)
  • Directional Lightを設定:
    • 良い影のために回転を調整
    • 推奨：回転(50, -30, 0)
    • 強度：1.0

├─ Environment
│  └─ Maze
├─ GameObjects
│  └─ Ball
├─ Lighting
│  └─ Main Light

  • オブジェクトを適切な親フォルダに整理
  • すべてのオブジェクトを明確にリネーム:
    • インポートした迷路 → 「Maze」

Step-by-Step Guide / ステップバイステップガイド

Detailed Workflow / 詳細ワークフロー

Part 1: Export from Blender (30 minutes)

  1. Open Blender and your Week 1 maze file
  2. Prepare the model:
    • Select all maze objects (A key)
    • Apply all transforms: Object → Apply → All Transforms
    • Check scale in properties: should be 1,1,1
  3. Export to FBX:
    • File → Export → FBX (.fbx)
    • Navigate to your Unity project location
    • Create folder: week-03/GameDev2025_Week03/Assets/Models/
    • Set export options:
      • Selected Objects: ON
      • Scale: 1.00
      • Forward: -Z Forward
      • Up: Y Up
      • Apply Scalings: FBX All
    • Save as: Maze_Week01.fbx

Part 2: Create Unity Project (15 minutes)

  1. Open Unity Hub
  2. Create New Project:
    • Click "New Project"
    • Template: 3D Core
    • Project name: GameDev2025_Week03
    • Location: Your GitHub repo/week-03/
    • Click "Create Project"
  3. Wait for Unity to open (first time may take a few minutes)
  4. Create folder structure in Project window:
    • Right-click in Assets → Create → Folder
    • Create: Models, Materials, Prefabs, Scenes

Part 3: Import and Build Scene (45 minutes)

  1. Import the maze (if not already there from export):
    • Drag Maze_Week01.fbx into Assets/Models folder
    • Unity will process the import
  2. Check import settings:
    • Click on Maze_Week01.fbx in Project window
    • Inspector → Model tab
    • Check Scale Factor (adjust if needed)
    • Click Apply
  3. Create new scene:
    • File → New Scene
    • File → Save As → Assets/Scenes/MazeGame.unity
  4. Add maze to scene:
    • Drag Maze_Week01.fbx from Project into Scene view
    • In Inspector, set Transform Position to (0, 0, 0)
    • Use F key to frame the maze in view
  5. Create ball:
    • Hierarchy → Right-click → 3D Object → Sphere
    • Rename to "Ball"
    • Inspector → Transform:
      • Position: Above maze start (e.g., 0, 1, 0)
      • Scale: 0.5, 0.5, 0.5
  6. Position camera:
    • Select "Main Camera" in Hierarchy
    • Inspector → Transform:
      • Position: (0, 15, -10)
      • Rotation: (45, 0, 0)
    • Adjust until you can see the whole maze in Game view
  7. Adjust lighting:
    • Select "Directional Light" in Hierarchy
    • Inspector → Transform → Rotation: (50, -30, 0)
    • Light component → Intensity: 1.0

Part 4: Organize Hierarchy (15 minutes)

  1. Create empty parent objects:
    • Hierarchy → Right-click → Create Empty
    • Rename to "Environment"
    • Repeat for "GameObjects" and "Lighting"
  2. Organize objects:
    • Drag "Maze" onto "Environment"
    • Drag "Ball" onto "GameObjects"
    • Drag "Directional Light" onto "Lighting"
    • Rename "Directional Light" to "Main Light"
  3. Verify hierarchy matches requirement

Part 5: Save and Submit (15 minutes)

  1. Save scene: File → Save (or Ctrl+S)
  2. Take screenshot:
    • Click on Game view tab
    • Take screenshot (Print Screen or Snipping Tool)
    • Save as week03_screenshot.png in week-03 folder
  3. Update .gitignore (if needed)
  4. Git commit and push:
    • Stage changes in GitHub Desktop
    • Commit message: "Complete Week 3: Import maze to Unity"
    • Push to GitHub
  5. Verify on GitHub.com that files are uploaded

Common Mistakes / よくある間違い

❌ Avoid These Mistakes:

1. Export Problems

  • Wrong orientation: Forgetting to set Forward=-Z, Up=Y
  • Wrong scale: Not applying transforms in Blender first
  • Missing objects: Not selecting all objects before export

2. Import Problems

  • Huge/tiny model: Scale Factor needs adjustment in import settings
  • Wrong location: Not placing FBX in Models folder
  • Forgot to click Apply: Changes to import settings not saved

3. Scene Setup Problems

  • Can't see maze: Camera position too close or wrong angle
  • Everything dark: Light intensity too low or wrong angle
  • Ball wrong size: Need to scale to match maze proportions

4. Organization Problems

  • Messy hierarchy: Not using parent folders
  • Bad names: Leaving default names like "Cube", "Sphere"
  • Scene not saved: Forgetting to save before closing Unity

5. GitHub Problems

  • Uploading Library folder: Wastes space, .gitignore not configured
  • Forgetting screenshot: Screenshot not in week-03 folder
  • Wrong project location: Unity project not in week-03 folder

Troubleshooting / トラブルシューティング

Problem: Maze appears upside down or sideways

Solution:

  • Re-export from Blender with correct Forward/Up settings
  • Or manually rotate in Unity: Select maze → Inspector → Rotation

Problem: Maze is way too big or way too small

Solution:

  • Select FBX in Project window
  • Inspector → Model tab → Scale Factor
  • Try: 0.01, 0.1, 1, 10, or 100
  • Click Apply after each change

Problem: Can't see anything in Scene view

Solution:

  • Select any object in Hierarchy
  • Press F key to frame it
  • Or double-click object name in Hierarchy

Problem: Game view is blank/black

Solution:

  • Make sure camera is positioned correctly
  • Check that camera is looking at the maze
  • Verify objects are in front of camera
  • Check lighting - add Directional Light if missing

Problem: Unity runs very slowly

Solution:

  • Close other programs
  • Lower quality settings: Edit → Project Settings → Quality
  • Disable real-time lighting in Scene view (sun icon)

Frequently Asked Questions / よくある質問

Q: Can I use a different maze if I didn't complete Week 1?

A: Yes, create a simple maze in Blender first (at least 5x5 units with some walls), then export and import it.

Q: Do I need to add materials/textures?

A: No, that's not required for Week 3. We'll cover materials later. The default white/gray appearance is fine.

Q: Can I use a different camera angle?

A: Yes! The suggested position is just a starting point. Position the camera however you think looks best for your maze.

Q: Should I add physics to the ball yet?

A: No, we'll add Rigidbody and physics in Week 4. For now, just place the ball in the scene.

Q: What if I can't install Unity on my computer?

A: Contact the instructor immediately. You may need to use a school computer or lab.

Q: How large should my Unity project be for GitHub?

A: With proper .gitignore, it should be under 50 MB. If larger, you're probably committing Library or Temp folders.

Q: Can I work with a classmate?

A: You can discuss concepts and help each other, but each student must submit their own maze and Unity project.
