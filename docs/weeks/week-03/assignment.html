Common Mistakes / ã‚ˆãã‚ã‚‹é–“é•ã„

âŒ Avoid These Mistakes:

1. Export Problems

  • Wrong orientation: Forgetting to set Forward=-Z, Up=Y
  • Wrong scale: Not applying transforms in Blender first
  • Missing objects: Not selecting all objects before export

2. Import Problems

  • Huge/tiny model: Scale Factor needs adjustment in import settings
  • Wrong location: Not placing FBX in Models folder
  • Forgot to click Apply: Changes to import settings not saved

3. Scene Setup Problems

  • Can't see maze: Camera position too close or wrong angle
  • Everything dark: Light intensity too low or wrong angle
  • Ball wrong size: Need to scale to match maze proportions

4. Organization Problems

  • Messy hierarchy: Not using parent folders
  • Bad names: Leaving default names like "Cube", "Sphere"
  • Scene not saved: Forgetting to save before closing Unity

5. GitHub Problems

  • Uploading Library folder: Wastes space, .gitignore not configured
  • Forgetting screenshot: Screenshot not in week-03 folder
  • Wrong project location: Unity project not in week-03 folder

Frequently Asked Questions / ã‚ˆãã‚ã‚‹è³ªå•

Q: Can I use a different maze if I didn't complete Week 1?

A: Yes, create a simple maze in Blender first (at least 5x5 units with some walls), then export and import it.

Q: Do I need to add materials/textures?

A: No, that's not required for Week 3. We'll cover materials later. The default white/gray appearance is fine.

Q: Can I use a different camera angle?

A: Yes! The suggested position is just a starting point. Position the camera however you think looks best for your maze.

Q: Should I add physics to the ball yet?

A: No, we'll add Rigidbody and physics in Week 4. For now, just place the ball in the scene.

Q: What if I can't install Unity on my computer?

A: Contact the instructor immediately. You may need to use a school computer or lab.

Q: How large should my Unity project be for GitHub?

A: With proper .gitignore, it should be under 50 MB. If larger, you're probably committing Library or Temp folders.

Q: Can I work with a classmate?

A: You can discuss concepts and help each other, but each student must submit their own maze and Unity project.
