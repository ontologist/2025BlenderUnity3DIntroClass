📥 Importing Maze from Blender

Export Settings (Blender):

  • File → Export → FBX
  • Scale: 1.00
  • Forward: -Z Forward
  • Up: Y Up
  • Apply Scalings: FBX All

Import to Unity:

  • Drag .fbx into Assets/Models
  • Check import scale
  • Add colliders to walls
  • Mark walls as Static

エクスポート設定（Blender）：

  • File → Export → FBX
  • Scale：1.00
  • Forward：-Z Forward
  • Up：Y Up
  • Apply Scalings：FBX All

Unityへのインポート：

  • .fbxをAssets/Modelsにドラッグ
  • インポートスケールを確認
  • 壁にコライダーを追加
  • 壁をStaticとしてマーク

🎯 Goal Detection System

How It Works:

  • Goal zone with trigger collider
  • OnTriggerEnter() detects player
  • Triggers win state in GameManager
  • Visual/audio feedback
  • Freezes ball movement

Implementation Steps:

  1. Create goal GameObject (cylinder/cube)
  2. Add Collider with Is Trigger ✓
  3. Add GoalTrigger script
  4. Check for "Player" tag
  5. Notify GameManager

仕組み：

  • トリガーコライダー付きゴールゾーン
  • OnTriggerEnter()でプレイヤーを検出
  • GameManagerで勝利状態をトリガー
  • 視覚/オーディオフィードバック
  • ボール移動を凍結

実装ステップ：

  1. ゴールGameObjectを作成（シリンダー/キューブ）
  2. Is Trigger ✓ でColliderを追加
  3. GoalTriggerスクリプトを追加
  4. 「Player」タグを確認
  5. GameManagerに通知

🎮 Game State Management

Game States:

  • Playing: Game is active
  • Won: Player reached goal
  • Lost: Player failed (fell off, etc.)
  • Paused: Game paused

GameManager Pattern:

  • Singleton design pattern
  • Manages state transitions
  • Controls UI visibility
  • Handles scene restart

ゲーム状態：

  • Playing： ゲームがアクティブ
  • Won： プレイヤーがゴールに到達
  • Lost： プレイヤーが失敗（落下など）
  • Paused： ゲームが一時停止

GameManagerパターン：

  • シングルトンデザインパターン
  • 状態遷移を管理
  • UIの可視性を制御
  • シーン再起動を処理

🔄 Scene Restart Functionality

Using SceneManager / SceneManagerの使用:

using UnityEngine.SceneManagement;

public void RestartLevel()
{
    SceneManager.LoadScene(
        SceneManager.GetActiveScene().buildIndex
    );
}

What it does:
- Reloads the current scene
- Resets all objects to initial state
- Called from Restart button
💡 Don't forget: Add scene to Build Settings!
💡 忘れずに： Build Settingsにシーンを追加！

⬇️ Fall Detection (Optional)

Detecting When Ball Falls:

  • Check Y position in Update()
  • If Y < threshold → trigger lose
  • Notify GameManager
  • Show lose UI panel

Implementation:

  • Add FallDetector script to ball
  • Set fall threshold (e.g., -10)
  • Call GameManager.OnBallFellOff()

ボールの落下を検出：

  • Update()でY位置を確認
  • Y < 閾値 → 敗北をトリガー
  • GameManagerに通知
  • 敗北UIパネルを表示

実装：

  • ボールにFallDetectorスクリプトを追加
  • 落下閾値を設定（例：-10）
  • GameManager.OnBallFellOff()を呼び出す

⚠️ Common Issues & Solutions

Physics Issues:

  • ❌ Ball jitters
  • ✅ Enable Interpolate
  • ❌ Ball gets stuck
  • ✅ Check collider sizes

Camera Issues:

  • ❌ Camera lags
  • ✅ Lower smoothSpeed
  • ❌ Camera through walls
  • ✅ Adjust offset

物理の問題：

  • ❌ ボールが揺れる
  • ✅ Interpolateを有効化
  • ❌ ボールが引っかかる
  • ✅ コライダーサイズを確認

カメラの問題：

  • ❌ カメラが遅れる
  • ✅ smoothSpeedを下げる
  • ❌ カメラが壁を通る
  • ✅ オフセットを調整

🖼️ UI Issues & Solutions

Common UI Problems:

  • ❌ UI not visible → Check Canvas Render Mode
  • ❌ Button doesn't work → Ensure EventSystem exists
  • ❌ Text not showing → Check Canvas Scaler settings
  • ❌ UI wrong size → Adjust Canvas Scaler

Solutions:

  • Canvas → Screen Space - Overlay
  • EventSystem auto-created with Canvas
  • Use Scale With Screen Size
  • Reference Resolution: 1920x1080

一般的なUIの問題：

  • ❌ UIが見えない → Canvas Render Modeを確認
  • ❌ ボタンが機能しない → EventSystemが存在することを確認
  • ❌ テキストが表示されない → Canvas Scaler設定を確認
  • ❌ UIサイズが間違っている → Canvas Scalerを調整

解決策：

  • Canvas → Screen Space - Overlay
  • EventSystemはCanvasで自動作成
  • Scale With Screen Sizeを使用
  • Reference Resolution：1920x1080
