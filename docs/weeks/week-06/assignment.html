Assignment Description / 課題概要

English: Implement AI enemy balls that chase the player using Unity's NavMesh system. Create intelligent enemies that navigate around obstacles and pursue the player, adding challenge to your ball maze game.

日本語: UnityのNavMeshシステムを使用してプレイヤーを追跡するAI敵ボールを実装します。障害物を回避し、プレイヤーを追跡する知的な敵を作成し、ボール迷路ゲームに挑戦を追加します。

Requirements / 要件

1. NavMesh Setup 3 points

  • Mark all walkable surfaces as Navigation Static
  • Bake NavMesh for your scene
  • NavMesh covers entire maze floor
  • Walls and obstacles excluded from NavMesh
  • NavMesh visible (blue) in Scene view
  • Configure agent radius (0.5-1.0 units) and height (1-2 units)

2. Enemy Ball GameObject 3 points

  • Create enemy ball prefab (Sphere primitive)
  • Apply contrasting material (red or different color)
  • Add Rigidbody component
  • Add NavMeshAgent component
  • Configure NavMeshAgent settings:
    • Speed: 3-5 units/second
    • Angular Speed: 120-180 degrees/second
    • Acceleration: 8 units/second²
    • Stopping Distance: 0.5 units
  • Tag enemy as "Enemy"
  • Set up appropriate layer for collision

3. Enemy Controller Script 5 points

  • Create EnemyController script
  • Find player GameObject by tag ("Player")
  • Set NavMeshAgent destination to player position
  • Update destination every frame in FixedUpdate()
  • Add detection range parameter (10-15 units)
  • Only chase player when within detection range
  • Stop chasing when player moves outside range
  • Code well-commented and organized

4. Collision Detection 2 points

  • Detect when enemy collides with player
  • Use OnTriggerEnter or OnCollisionEnter
  • Call GameManager.GameOver() when collision occurs
  • Ensure player ball has "Player" tag
  • Add visual/audio feedback (optional but recommended)

5. Testing and Balancing 2 points

  • Test enemy chases player correctly
  • Verify enemy navigates around obstacles
  • Ensure enemy doesn't get stuck
  • Adjust enemy speed to feel challenging but fair
  • Adjust detection range for good gameplay feel
  • Document chosen values in code comments

Code Template / コードテンプレート

using UnityEngine;
using UnityEngine.AI;

public class EnemyController : MonoBehaviour
{
    private NavMeshAgent agent;
    private Transform player;
    public float detectionRange = 10f;
    public float chaseRange = 15f;
    
    void Start()
    {
        agent = GetComponent<NavMeshAgent>();
        player = GameObject.FindGameObjectWithTag("Player").transform;
    }
    
    void FixedUpdate()
    {
        if (player != null)
        {
            float distance = Vector3.Distance(
                transform.position, player.position);
            
            if (distance <= detectionRange)
            {
                agent.SetDestination(player.position);
            }
            else if (distance > chaseRange)
            {
                agent.ResetPath();
            }
        }
    }
    
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            GameManager.Instance.GameOver();
        }
    }
}

1. Multiple Enemy Types (+1 point)

2. Enemy Spawner (+1 point)

Create EnemySpawner script that spawns enemies at specific locations, every X seconds, with maximum enemy count limit.

3. Advanced AI Behavior (+1 point)

Implement patrol behavior when player not detected, add enemy health system, or create spawn points as GameObjects with visual indicators.

  • Enemy prefab created and configured
  • EnemyController script works correctly
  • Enemy chases player when in range
  • Collision detection triggers game over
  • Enemy doesn't get stuck on obstacles
  • Game feels balanced and playable

  • Screenshot of NavMesh included (week06_navmesh_screenshot.png)

Common Mistakes to Avoid / 避けるべき一般的な間違い

  • ❌ Forgetting to bake NavMesh (enemy won't move)
  • ❌ Not updating destination every frame
  • ❌ Wrong NavMeshAgent size (gets stuck)
  • ❌ Not setting Player tag on player ball
  • ❌ Using wrong collision detection method
  • ❌ Enemy speed too fast or too slow
  • ❌ Detection range too large (unfair) or too small (too easy)
